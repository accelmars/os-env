## Manual Steps Before First Release

### Gate 4 — Branch Protection

Run in terminal:
```bash
gh api repos/accelmars/resolver-env/branches/main/protection \
  --method PUT \
  --field required_status_checks='{"strict":true,"contexts":["ci"]}' \
  --field enforce_admins=false \
  --field required_pull_request_reviews=null \
  --field restrictions=null
```

Or in GitHub UI: **Settings → Branches → Add rule**:
- Branch name pattern: `main`
- ✅ Require status checks to pass before merging
  - Required check: `ci`
- ✅ Require branches to be up to date before merging
- ✅ Do not allow bypassing the above settings (recommended)

### Gate 5 — Initial Tag

```bash
cd /path/to/resolver-env
git tag accelmars-resolver-env-v0.1.0
git push origin accelmars-resolver-env-v0.1.0
```

This is required before release-plz can compute the next version. Without an initial tag, release-plz has no baseline to diff from.

### Gate 3 — GitHub Secrets

In **Settings → Secrets and variables → Actions**, add:
- `RELEASE_PLZ_TOKEN` — a fine-grained PAT with `contents: write` and `pull-requests: write` on this repo
- `CARGO_REGISTRY_TOKEN` — crates.io token (if you ever publish; harmless to add now even with `publish = false`)
